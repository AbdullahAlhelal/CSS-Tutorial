Welcome to the lesson on the CSS cascade! This page demonstrates how multiple CSS rules can target the same element and how the final style is determined by factors such as the origin of the styles, specificity, order of appearance, and the use of !important.

Example 1: Basic Cascade and Specificity

This paragraph is styled by multiple rules:
- The element selector sets the color to blue.
- The class selector (.example) changes the color to green and increases the font size.

Example 2: ID Selector Overrides Class and Element

This paragraph has both an ID ("unique") and a class ("example").
The ID selector, being more specific, overrides the class and element selectors. The final text color is purple and the font size is 20px.

Example 3: Inline Style Wins

This paragraph has an inline style that sets the text color to red and font size to 22px.
Inline styles have the highest specificity, so these values override the ID, class, and element selectors.

Example 4: Order of Appearance

When two rules have the same specificity, the one that appears later in the stylesheet wins.
In our CSS, the second p rule (adding a border) is applied after the first one, ensuring that every paragraph gets a black border.

Example 5: Using !important

This paragraph uses the class "important", which applies the style color: orange !important.
The !important declaration forces the color to be orange, overriding even inline styles or more specific selectors for color.

Summary

The cascade in CSS is the process by which the browser determines which styles to apply. It takes into account:

  • Origin of the Styles: (user, author, or user-agent stylesheets)
  • Specificity: Inline styles > IDs > Classes/attributes/pseudo-classes > Elements/pseudo-elements
  • Order of Appearance: Later rules override earlier ones if specificity is equal
  • Importance: The !important declaration forces a rule to override others
